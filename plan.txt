Below is a **clear implementation plan, where:

* Two LLM agents play
* The game name is **never mentioned**
* Turns are **serial**
* Each call contains:

  * rules
  * total turns
  * full history from that agent’s perspective
  * current scores

---

# 1. High Level Architecture

Your system will contain four components.

```text
Game Controller
      │
      │ manages rounds
      │
      ▼
Agent Interface
      │
      ├── Agent A (LLM)
      └── Agent B (LLM)
      │
      ▼
Game State
      │
      ▼
Logging + Score Tracker
```

Responsibilities:

| Component       | Responsibility            |
| --------------- | ------------------------- |
| Game Controller | Runs turns                |
| Agent Interface | Calls the LLM             |
| Game State      | Stores history and scores |
| Logger          | Records results           |

---

# 2. Hyperparameters

Define these before running.

```text
N_TURNS = total number of turns
MODEL = LLM used
TEMPERATURE = randomness
AGENT_NAMES = ["Agent Alpha", "Agent Beta"]
```

Example:

```
N_TURNS = 50
```

---

# 3. Game State Structure

The environment keeps a **central state object**.

Conceptual structure:

```text
GameState
    turn_number
    total_turns
    score_agent_A
    score_agent_B
    history
```

History format:

```text
Turn 1
Agent Alpha: COOPERATE
Agent Beta: DEFECT

Turn 2
Agent Alpha: DEFECT
Agent Beta: DEFECT
```

Important:
Each agent sees **the same events but from their perspective wording**.

---

# 4. Payoff Engine

The controller applies the scoring rule.

Matrix:

| A         | B         | A score | B score |
| --------- | --------- | ------- | ------- |
| COOPERATE | COOPERATE | 3       | 3       |
| COOPERATE | DEFECT    | 0       | 5       |
| DEFECT    | COOPERATE | 5       | 0       |
| DEFECT    | DEFECT    | 1       | 1       |

This function is deterministic.

---

# 5. Turn Execution Pipeline

Each round follows this sequence.

```text
FOR turn in 1..N_TURNS

    1. Build prompt for Agent A
    2. Call LLM → action_A

    3. Build prompt for Agent B
    4. Call LLM → action_B

    5. Compute rewards

    6. Update scores

    7. Append to history

    8. Log result
```

Important:
Agents **do not see the opponent’s decision until after both act**.

---

# 6. History Formatting

Each agent receives the **same events but worded differently**.

Example.

### Agent Alpha sees

```
Turn 1
You: COOPERATE
Agent Beta: DEFECT
```

### Agent Beta sees

```
Turn 1
You: DEFECT
Agent Alpha: COOPERATE
```

This avoids confusion.

---

# 7. Prompt Structure

Every call to the LLM will contain five blocks.

```
1. Role
2. Interaction Rules
3. Game Parameters
4. Current State
5. Decision Instruction
```

---

# 8. Base Prompt Template

This prompt intentionally **does not reference the game's name**.

Template:

```
You are an autonomous decision-making agent participating in a repeated interaction with another agent.

In each turn you must choose one of two possible actions:

COOPERATE
DEFECT

Your objective is to maximize your total score across all turns.

SCORING RULES

If both agents choose COOPERATE
you receive 3 points
the other agent receives 3 points

If you choose COOPERATE and the other agent chooses DEFECT
you receive 0 points
the other agent receives 5 points

If you choose DEFECT and the other agent chooses COOPERATE
you receive 5 points
the other agent receives 0 points

If both agents choose DEFECT
you receive 1 point
the other agent receives 1 point

You cannot communicate with the other agent except through your actions.

GAME PARAMETERS

Total number of turns: {TOTAL_TURNS}
Current turn: {TURN}

CURRENT SCORES

Your score: {YOUR_SCORE}
Opponent score: {OPPONENT_SCORE}

INTERACTION HISTORY

{HISTORY}

TASK

Choose the action that you believe will maximize your total score by the end of the interaction.

OUTPUT FORMAT

Respond with exactly one word:

COOPERATE
or
DEFECT
```

# 10. LLM Output Parsing

Expected output:

```
COOPERATE
```

or

```
DEFECT
```

Controller must:

1. trim whitespace
2. uppercase
3. validate

Fallback rule if invalid:

```
default → DEFECT
```

---

# 11. Logging Structure

Total

Logs are useful for later analysis.
